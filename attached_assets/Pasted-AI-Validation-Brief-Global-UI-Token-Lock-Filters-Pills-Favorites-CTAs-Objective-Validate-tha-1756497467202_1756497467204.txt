AI Validation Brief — Global UI Token Lock (Filters, Pills, Favorites, CTAs)
Objective

Validate that the locked global tokens are applied and inherited consistently across the app (no changes to UI; verification only).

Governance (Do NOT change anything)

Visual verification only — do not alter styles, tokens, or layout.

If a mismatch is found, report it; do not “hot-fix”.

Scope: Category pills, “Newest First” dropdown, Favorites button, Primary CTAs.

What to Verify (Computed Styles)

For each component, capture computed CSS (browser devtools) and confirm:

A) Category Pills

Height = 36px; Border-radius = 12px

Inactive: background #F8F9FC, border #E5E7EB, shadow = --token-shadow-secondary

Active: background = --token-gradient-primary, text #FFF, shadow = --token-shadow-primary

Hover: scale ≈ 1.05, primary or secondary shadow increases

B) “Newest First” Dropdown

Height = 36px; Radius = 12px

Neutral surface (no gradient), border #E5E7EB

Hover: subtle scale/halo; no gradient

C) Favorites Button

Height = 36px; Radius = 12px

Neutral surface, border #E5E7EB, shadow = secondary

Badge = 24×24px, background = --token-badge-danger, centered text

D) Primary CTAs

Height = 44px (intentionally taller)

Background = --token-gradient-primary

Shadow = primary; hover = primary-hover

E) Alignment

In the filter bar, category pills, “Newest First”, and Favorites share the same top baseline and visual center.

“CATEGORIES” label does not push pills down.

F) Responsiveness (3 widths)

≥1200px desktop: one line; aligned

768–1199px tablet: stack/wrap gracefully; heights unchanged

≤767px mobile: wrap chips; heights unchanged; badge readable

G) Accessibility

Focus ring visible on all interactive elements (matches --token-focus-ring)

Text contrast meets WCAG AA (pill text, dropdown text, favorites badge)
