What to keep

Do not edit the product card component or the product details modal.

Keep the existing filter chips (incl. Instagram/Facebook icons), search box, and sort.

Keep the current neutral light theme; we’ll layer brand accents only in the header/CTAs.

Guardrails (to avoid regressions)

Never style raw tags (h1, img, button) globally. Scope all page styles under a page wrapper class.

Lock layout width: use a single container on every app page.

Define a consistent card grid and never override the card’s internal layout.

2) Reusable page shell (scoped; won’t touch cards)

Create a tiny wrapper so every page gets the same container & spacing. Do not change card markup.

// components/PageShell.tsx
export default function PageShell({
  title,
  subtitle,
  actions,
  children,
}: { title: string; subtitle?: string; actions?: React.ReactNode; children: React.ReactNode }) {
  return (
    <div className="bg-gradient-to-b from-slate-50 to-white">
      <div className="mx-auto max-w-7xl px-6 pt-8 pb-4">
        <div className="flex items-start justify-between gap-4">
          <div>
            <h1 className="text-3xl md:text-4xl font-extrabold tracking-tight text-slate-900">{title}</h1>
            {subtitle && <p className="mt-2 text-slate-600">{subtitle}</p>}
          </div>
          {actions}
        </div>
        {/* toolbar slot */}
      </div>
      {children}
    </div>
  );
}

4) Keep card media uniform (if you see mismatched heights)

If—and only if—some cards wobble, wrap the image container (not the img itself) with a fixed ratio:

<div className="aspect-[16/9] overflow-hidden rounded-t-2xl bg-slate-100">
  <img className="h-full w-full object-cover" ... />
</div>


Do not add global img rules.

Visual polish checklist (non-breaking)

Header bar: leave as-is; apply brand gradient only to “Add Product” and primary CTAs.

Title scale: text-3xl md:text-4xl (don’t exceed).

Spacing rhythm: container blocks use px-6, gap-6, and rounded-2xl.

Shadows: use shadow-soft on glass/toolbar, not on cards (cards already look good).

Motion: keep hover transitions to opacity/translate-y-0.5 on buttons only; cards unchanged.

Responsiveness: grid 1/2/3 columns as above; keep max width at max-w-7xl.

Why the last change broke things

You likely removed max-w-* containers and/or applied global typography or image styles. That made the cards full-bleed and scaled their media.

The fix above reintroduces scoped layout and glass toolbar while leaving card internals untouched.
