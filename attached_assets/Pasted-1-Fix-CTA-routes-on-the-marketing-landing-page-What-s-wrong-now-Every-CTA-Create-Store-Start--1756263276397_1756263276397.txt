What we want

2) Guard onboarding routing (Step 2 only when Step 1 is complete)

What’s wrong now
Users can deep-link or be pushed directly to Step 2.

What we want

If a user hits Step 2 without Step 1 data present, redirect to Step 1.

When a user clicks Create store (or any marketing CTA), clear any previous onboarding state so the wizard always starts on Step 1.

Implementation hints:

If you store Step 1 data in a context, Zustand store, or localStorage/sessionStorage, add a simple check in Step 2:

// In Onboarding Step 2 component
const hasStep1 = useOnboardingStore(s => s.basicInfoComplete); // or equivalent

useEffect(() => {
  if (!hasStep1) navigate('/onboarding?step=1', { replace: true });
}, [hasStep1]);


When entering from a marketing CTA, reset the wizard:

// in landing CTA click handler or a small hook fired on landing→onboarding transition
sessionStorage.removeItem('onboarding'); // or call your store.reset()


If you’re using query param step, the route /onboarding?step=1 should always render Step 1 and set the guard for step 2+.
