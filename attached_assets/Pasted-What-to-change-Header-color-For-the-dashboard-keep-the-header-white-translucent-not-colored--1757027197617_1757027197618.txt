What to change

Header color

For the dashboard, keep the header white / translucent (not colored). It lets the gradient CTAs and glass surfaces carry the brand without fighting the content.

Style: bg-white/80 backdrop-blur supports-[backdrop-filter]:bg-white/60 border-b border-slate-200.

Keep icons/nav text slate (600 default → 900 on active).

Logo

The tiny bug mark at the far left looks out of place. Either:

Use the wordmark only, or

Use a single, monochrome bug sized to the wordmark’s cap height (≈18–20px) with consistent spacing (8–12px) before “ShopLynk.”

If you don’t have the final SVG handy, use a text wordmark for now (below) and remove the extra mini logo.

Spelling / capitalization

It must be ShopLynk (capital S + capital L).

Fix anywhere it appears as ShoplYnk, ShopLynk with wrong case, or mixed-case variants.

Also update <title>, aria-label, and alt text to “ShopLynk”.

Quick checklist for the dev

 Header uses white/blur with subtle bottom border; no colored header bar.

 Remove the misfitting mini logo; use wordmark (or a single, sized bug + wordmark).

 All instances read ShopLynk (capital L).

 Active nav item = darker text or light slate pill background; hover states consistent.

 Gradient CTAs (like Add Product) use the landing’s sky→violet gradient; chips/badges remain neutral.
